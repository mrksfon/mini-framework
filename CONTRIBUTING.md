# Contributing

Thank you for considering contributing to Mini Framework.

## Workflow

1. Create or choose an issue.
2. Create a branch from `main`.
3. Keep the change focused on a single issue.
4. Add or update tests.
5. Run the quality checks.
6. Open a pull request.

## Branch Names

Use one of these prefixes:

- `feat/` for features
- `fix/` for bug fixes
- `docs/` for documentation
- `chore/` for tooling and maintenance

## Quality Checks

Before opening a pull request, run:

```bash
composer check

```

## Pull Requests

Describe what changed and why. Link the related issue using:

```text
Closes #123
```
